# Wedding Invitation — Theatre Style

## Project Overview
Invitation de mariage digitale interactive avec un thème "rideau de théâtre". Site one-page mobile-first, scroll vertical, avec animations fluides et interactions tactiles (scratch-to-reveal). Destiné à être partagé par lien (WhatsApp, Instagram, SMS). Bilingue français / arabe.

## Stack
- **Framework** : React 18 + Vite + TypeScript
- **Styling** : Tailwind CSS 3
- **Animations** : Framer Motion
- **Canvas** : HTML5 Canvas API (scratch effect)
- **Audio** : HTML5 Audio API (musique de fond MP3)
- **Deployment** : Cloudflare Pages
- **Package manager** : npm

## Internationalisation (FR/AR)

### Principe
- Système i18n léger et custom (pas de lib externe, le site est petit)
- Un `LanguageContext` avec `useTranslation()` hook
- Toggle FR / عربي en haut à droite de l'écran, toujours visible (position fixed)
- Le toggle est un pill button avec les deux options, l'option active est colorée (fond `--curtain`, texte blanc)

### Direction du texte
- FR : `dir="ltr"`, texte aligné center
- AR : `dir="rtl"`, texte aligné center (centré pour une invitation)
- Appliquer `dir` sur le `<html>` tag dynamiquement via un `useEffect`
- Les prénoms restent TOUJOURS en LTR même en mode arabe (les prénoms ne changent pas de sens)

## Audio

### Musique de fond
- Fichier MP3 placé dans `public/music.mp3` (ajouté manuellement par l'utilisateur)
- Chemin configurable dans `wedding.ts` : `musicUrl: "/music.mp3"`
- La musique ne démarre PAS automatiquement (les navigateurs bloquent l'autoplay)
- La musique démarre au premier tap de l'utilisateur sur le bouton son
- Hook `useAudio.ts` :
  - `play()`, `pause()`, `toggle()`, `isPlaying: boolean`
  - Gérer le cas où l'audio est bloqué par le navigateur (`play()` retourne une Promise)
  - Volume par défaut : `0.3`
  - Loop activé : `audio.loop = true`

### Bouton AudioPlayer
- Position fixe, bas à droite, `z-index: 50`
- Cercle brun foncé `#4A1A1A` (~50px diamètre)
- Icône SVG inline : haut-parleur avec ondes si playing, haut-parleur barré si paused
- Légère animation de pulsation CSS quand la musique joue

## Conventions de code

### TypeScript
- Strict mode activé
- Interfaces pour tous les props de composants
- Pas de `any`
- Constantes pour les configs

### React
- Functional components uniquement avec hooks
- Chaque composant dans son propre fichier
- Props typées avec interface `XxxProps`
- Pas de inline styles — tout en Tailwind ou classes CSS custom

### Tailwind
- Classes utilitaires Tailwind au maximum
- CSS custom uniquement pour : clip-paths, gradients complexes, animations de rideau, RTL overrides
- Couleurs custom dans `tailwind.config.ts` via `extend.colors`
- Font families custom dans `extend.fontFamily`

### Animations
- Framer Motion pour : ouverture rideau, fade-in au scroll, confettis
- CSS `@keyframes` pour : lignes du rideau, pulsation bouton son
- `will-change` et `transform` pour la performance GPU
- Durées : transitions rapides ~300ms, rideau ~1.5s, confettis ~2s

### Canvas (Scratch)
- `globalCompositeOperation: 'destination-out'` pour l'effet grattage
- Supporter touch events (`touchstart`, `touchmove`) ET mouse events (`mousedown`, `mousemove`)
- Déclencher le reveal quand ~60% du cercle est gratté
- Texture dorée = gradient radial CSS en dessous du canvas

## Sections du site (ordre de scroll)

1. **Rideau fermé** — 100vh, fond rouge, lignes verticales fines ondulées
2. **Rideau ouvert + Invitation** — animation d'ouverture (2 rideaux s'écartent en V), prénoms en cursive, message
3. **Scratch Reveal** — 3 cercles dorés (canvas scratch), confettis quand tous grattés, "On se marie !"
4. **Countdown** — 4 boîtes bordées (JOURS, HEURES, MIN, SEC) timer live
5. **Venue** — lieu, illustration placeholder, lien Google Maps
6. **RSVP** — formulaire (nom, nb personnes, message), endpoint configurable

## RSVP

### Formulaire
- Nom complet (input text, required)
- Nombre de personnes (select 1-5, required)
- Message (textarea, optional)
- Bouton submit : fond `--curtain`, texte blanc, hover `--curtain-dark`

### Backend
- Endpoint configurable dans `wedding.ts` : `rsvpEndpoint: ""`
- Si endpoint vide → mode démo (affiche succès sans envoyer)
- Si endpoint rempli → `POST` avec `{ name, guests, message, language }` en JSON
- États : idle, loading, success, error
- Compatible avec : Google Sheets API, Supabase, n8n webhook, tout endpoint REST

```typescript
rsvp: {
  endpoint: "",
  method: "POST",
}
```

## Performance
- Rideaux en CSS/SVG (pas d'images lourdes)
- Lazy-load sections avec Intersection Observer
- Canvas scratch actif uniquement quand visible
- Fonts arabes chargées en async
- MP3 chargé en lazy
- Lighthouse cible : 90+ performance mobile
